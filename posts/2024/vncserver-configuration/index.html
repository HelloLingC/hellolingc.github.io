废弃的Android设备成为BT下载服务器

August 3, 2024 · 1 分钟 · LingC

讲道理，我还从来没关注过VNC这个东西。早在几年前就知道了VNC，但其实我一直都用不上。

最近，我的Android设备因为电池寿命问题已经终止使用了，我就打算让他成为一个永久的·被插入的状态（指充电）。

为了方便，我直接用傻瓜脚本在Termux中装上Ubuntu: https://github.com/MFDGaming/ubuntu-in-termux

首先先安装ssh，方便直接在电脑中输入命令。

$ sudo apt install openssh-server
$ systemctl status ssh

为了让VNC客户端能够连接到手机中的Ubuntu，要先装一个桌面。

$ sudo apt install ubuntu-desktop

我选择的是TightVNC，其他的还有TigerVNC什么的，我只是随便选了一个。

$ apt install tightvncserver

就在我毫无感情地用vncserver指令启动时，提示我环境变量 $USER 没有设置。

啊？

$ export USER=root
$ source ~/.bashrc

任何奇怪的东西只要遇见Termux就都能接受。

然而我从电脑中的VNC Viewer连接是灰屏的，什么也没有。

然后我瞎🐔把改了改~/.vnc/xstartup，好消息是不再灰屏了，

坏消息是变成只显示File Explorer的状态了，其他桌面上什么也没有。我实在是不知道怎么配置VNC啊……

当我看到 localhost:1 的时候，我两眼一黑，后来才知道VNC中端口是5900+X, X=1,2,3…

vncserver -kill :1
